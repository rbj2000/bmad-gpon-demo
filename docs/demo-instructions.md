# GPON Synthetic Data Demo

Generated by: migration-pipeline-orchestrator | Date: 2026-02-19

## What This Demo Does

Generates synthetic GPON access network data and loads it into **NetBox** (REST) and/or **Kuwaiba** (SOAP) target inventory systems running in Docker.

## Prerequisites

- Python 3.10+
- Docker and Docker Compose
- For NetBox: `pip install requests pyyaml`
- For Kuwaiba: `pip install zeep pyyaml`
- PyYAML is optional (a built-in fallback parser handles the demo config)

## Quick Start: NetBox

```bash
# 1. Start NetBox (first startup takes ~60s)
docker compose -f scripts/docker-compose.netbox.yml up -d

# 2. Wait for NetBox to be ready (v4.x requires auth for all API endpoints)
curl -s http://localhost:8000/api/status/ \
  -H "Authorization: Token 0123456789abcdef0123456789abcdef01234567" \
  | python3 -m json.tool

# 3. Run the demo
python staging/demo/run_demo.py --target netbox

# 4. Verify in browser
open http://localhost:8000
# Login: admin / admin
# Check: Devices > Devices (should see OLTs, splitters, ONTs)
```

## Step-by-Step (Manual)

### 1. Generate Synthetic Data

```bash
cd staging/scripts/synthetic
python generate_synthetic_data.py --config config.yaml --output-dir ./output --seed 42
```

Output in `staging/scripts/synthetic/output/`:
- `sites.csv` — 5 sites
- `olts.csv` — 10 OLTs
- `slots.csv` — ~40 chassis slots
- `line_cards.csv` — ~40 GPON/control line cards
- `splitters.csv` — 40 splitters
- `sfp_transceivers.csv` — ~40 SFP transceivers
- `onts.csv` — 200 ONTs
- `connections.csv` — 240 fiber links
- `manifest.json` — file list with row counts and SHA-256 checksums

### 2. Load into NetBox

```bash
cd staging/scripts/load
python load_data.py --adapter netbox --config load-config.yaml --data-dir ../synthetic/output
```

### 3. Load into Kuwaiba

```bash
cd staging/scripts/load
python load_data.py --adapter kuwaiba --config load-config.yaml --data-dir ../synthetic/output
```

### 4. Dry Run (No API Calls)

```bash
python load_data.py --adapter netbox --config load-config.yaml --data-dir ../synthetic/output --dry-run
```

## Script Reference

### `generate_synthetic_data.py`

| Flag           | Default        | Description                          |
|----------------|----------------|--------------------------------------|
| `--config`     | `config.yaml`  | Path to entity config YAML           |
| `--output-dir` | from config    | Override output directory             |
| `--seed`       | from config    | Random seed for reproducibility      |

### `load_data.py`

| Flag          | Default                | Description                        |
|---------------|------------------------|------------------------------------|
| `--adapter`   | *(required)*           | `netbox` or `kuwaiba`              |
| `--config`    | `load-config.yaml`     | Path to load configuration         |
| `--data-dir`  | `../synthetic/output`  | Directory with generated CSVs      |
| `--dry-run`   | off                    | Log operations without creating    |
| `--fail-fast` | off                    | Stop on first error                |

### `run_demo.py`

| Flag              | Default | Description                           |
|-------------------|---------|---------------------------------------|
| `--target`        | *(required)* | `netbox`, `kuwaiba`, or `both`   |
| `--generate-only` | off     | Only generate data, don't load        |
| `--dry-run`       | off     | Generate data, simulate loading       |
| `--seed`          | `42`    | Random seed for generation            |

## Troubleshooting

### Target system not running

```
ERROR: netbox is not reachable
```

Start the target:
```bash
docker compose -f scripts/docker-compose.netbox.yml up -d
# Wait 60s for NetBox, 30s for Kuwaiba
```

### Authentication errors (NetBox)

```
401 Unauthorized
```

Check that the token in `load-config.yaml` matches the one in `docker-compose.netbox.yml`:
```yaml
# REVIEW: Both files must use the same token value
token: "0123456789abcdef0123456789abcdef01234567"
```

### Containment errors (Kuwaiba)

```
ERROR: Containment error creating OLT
```

Kuwaiba requires containment rules allowing parent→child relationships.
The demo uses a simplified hierarchy (City → OLT → Splice → ONT).

<!-- REVIEW: Production deployments need the full hierarchy:
     City → Building → Floor → Room → Rack → OLT → Splice → ONT
     Add containment rules via Kuwaiba's admin UI. -->

### Duplicate records on re-run

- **NetBox**: The adapter uses `_get_or_create` for model setup and handles 409 (duplicate) on cables. Devices are always created new — to re-run cleanly, delete existing demo data first.
- **Kuwaiba**: SOAP `createObject` may fail on duplicates. Delete previous demo objects or use a fresh container.

### Teardown

```bash
# Stop and remove NetBox
docker compose -f scripts/docker-compose.netbox.yml down -v

# Stop and remove Kuwaiba
docker compose -f scripts/docker-compose.kuwaiba.yml down -v
```
